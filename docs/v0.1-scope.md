# v0.1 Scope

AgentScienceLab v0.1 is a public-docs and configuration-readiness step for a research-team configuration layer on AiPlus.

## Included

v0.1 includes:

- README positioning for a public repository
- docs for positioning, non-goals, roles, aliases, scope, and AiPlus alignment
- narrative examples for experiment design review, evaluation audit, and reproducibility checks
- consultant policy aligned to AgentScienceLab research roles
- research role descriptions for claim discipline, evaluation review, reproducibility, literature positioning, critique, and writing support

## Excluded

v0.1 excludes:

- benchmark runner
- benchmark suite
- leaderboard infrastructure
- autonomous scientist loop
- CLI wrapper
- runtime roster replacement
- runtime adapter changes
- agent framework behavior
- substrate behavior changes
- secret broker changes
- deploy, release, package, or external-account automation

## Known Role ID Naming Gap

Role ID shape is resolved locally with kebab-case canonical IDs, such as `research-lead` and `experiment-designer`, while awaiting AiPlus upstream proposal alignment.

The ASL canonical role IDs are:

- `research-lead`
- `advisor`
- `experiment-designer`
- `benchmark-engineer`
- `evaluation-scientist`
- `agent-systems-engineer`
- `reproducibility-engineer`
- `literature-reviewer`
- `referee`
- `paper-writer`

The ASL-local `.aiplus/consultant-team.toml` has been synchronized to these canonical IDs, and `docs/roles.md` includes canonical ID notes under each display-name heading. The remaining alignment task is an AiPlus-side proposal update so future module asset names match the ASL repository.

## Substrate Delegation

All substrate behavior delegates to AiPlus. AgentScienceLab does not define independent semantics for routing, memory, dispatch, locks, worktrees, lanes, MCP, runtime launch, status, refresh, safety, secret handling, or Owner gates.

If a requested feature requires new substrate behavior, the default v0.1 answer is to document the research workflow need, route the substrate question to AiPlus, or defer the feature until explicit Owner approval.
