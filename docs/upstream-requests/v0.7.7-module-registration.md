# AiPlus Upstream Request: v0.7.7 ASL Module Registration

Status: pending upstream review
Owner: izhiwen
ASL-side ref: commit d75c9a3 (github.com/izhiwen/AgentScienceLab @ main)
Target: AiPlus core (aiplus-public, crates/aiplus-core/src/)
Estimated effort: ~1 hour Rust work + tests

## Problem

AgentScienceLab (ASL) is an AEL-clone-pattern AiPlus module. ASL's own `scripts/build-asl.sh` syncs
the ASL repository into `vendor/aiplus/assets/agentsciencelab/` and rebuilds the aiplus binary so ASL
assets are bundled. Build succeeds, but the resulting binary does not recognize `agentsciencelab` as
a known module because AiPlus core's module registry is a static array of hardcoded `ModuleSpec`
entries, and ASL is not in that array.

Result: `aiplus add agentsciencelab` returns "no such module"; `./asl install --runtime claude-code`
fails with `bundled runtime not built` or "module not found" depending on where the dispatch lands.

## ASL-side state already in place (commit d75c9a3)

- Repo: github.com/izhiwen/AgentScienceLab
- `aiplus-module.json` at root: `name="agentsciencelab"`, `abbreviation="ASL"`, `version="0.1.0"`
- 10 canonical roles (kebab-case): pi, advisor, experiment-designer, benchmark-engineer,
evaluation-scientist, agent-systems-engineer, reproducibility-engineer, literature-reviewer, referee,
paper-writer
- `core/templates/` has 10 role TOMLs + 10 personas + `agent-science-team.toml` +
`consultant-team.agentsciencelab.toml`
- `adapters/` has claude-code / codex / opencode subagent manifests with `asl-` prefixed slash
commands
- `scripts/build-asl.sh` syncs ASL repo into `vendor/aiplus/assets/agentsciencelab/` before `cargo
build`
- `asl` binary (bash wrapper) is aligned: dispatch + help + version all reflect ASL canonical 10
roles

ASL is structurally parallel to AEL (github.com/izhiwen/AiEconLab). All ASL repo infrastructure is
ready. The only blocker is the missing registration in AiPlus core.

## Desired AiPlus capability

Register `agentsciencelab` as a bundled, opt-in module in AiPlus core, paralleling the existing
`aieconlab` entry. After the change, `aiplus add agentsciencelab` should install the module, and
`aiplus agent list` should show ASL canonical roles.

## Why ASL cannot solve this locally

- `module_manifest.rs` is AiPlus source; ASL Advisor/CEO boundary forbids modifying AiPlus repository
- The bundled module registry is a static Rust array (`const MODULES: &[ModuleSpec]`); it cannot be
extended at runtime by an external module
- ASL's `scripts/build-asl.sh` already does the asset sync correctly (writes to
`vendor/aiplus/assets/agentsciencelab/`); the only gap is that AiPlus does not know to look for that
path under that name
- The 5 changes above are purely additive (parallel to existing `aieconlab` pattern), do not modify
or remove any existing AiPlus behavior, and follow the established opt-in module convention
