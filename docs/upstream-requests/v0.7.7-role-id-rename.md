# ASL To AiPlus Upstream Request: v0.7.7 Role ID Rename

Status: request for AiPlus-side docs alignment

## Problem

The AiPlus-side proposal `docs/proposals/v0.7.7-asl-module-onboarding.md` uses `pi` for the research lead role and kebab-case IDs for other AgentScienceLab roles. The AgentScienceLab repository has now standardized locally on kebab-case canonical IDs with explicit `research-lead` instead of `pi`.

This creates a naming mismatch between the public ASL repository and the future AiPlus module onboarding proposal.

## Desired AiPlus Capability

Align the upstream AgentScienceLab module onboarding proposal role IDs with the ASL canonical IDs before module assets, runtime adapter names, worktree names, or memory directory names are implemented.

## ASL-Side Evidence

ASL canonical IDs are documented in:

- `docs/roles.md`
- `.aiplus/consultant-team.toml`
- `docs/aliases.md`

The canonical IDs are:

- `research-lead`
- `advisor`
- `experiment-designer`
- `benchmark-engineer`
- `evaluation-scientist`
- `agent-systems-engineer`
- `reproducibility-engineer`
- `literature-reviewer`
- `referee`
- `paper-writer`

## Minimal Requested AiPlus Change

Update the AiPlus-side proposal text only. No implementation change is required for this request.

Specifically, update `docs/proposals/v0.7.7-asl-module-onboarding.md` so the proposed role IDs match the ASL canonical IDs listed above.

## Why ASL Cannot Solve This Locally

The proposal lives in the AiPlus repository and governs future module asset names, runtime adapter names, and onboarding expectations. The ASL repository can document its canonical IDs, but it cannot update the AiPlus-side proposal from here without an upstream AiPlus change.
